# Meaning Preservation System

## Status Labels

- Live verified: not live verified.
- Merged source: not implemented as a system.
- Prototype: not a software prototype.
- Proposed: proposed internal governance system.
- Planned: may become a documentation and review practice.
- Not yet available: not available as a public feature, AI layer, or automated memory system.

## Purpose

Meaning Preservation is the internal discipline of keeping original intent, evidence, uncertainty, and boundaries intact as ideas move from founder notes to docs, code, public copy, pilots, and reports.

## System Components

| Component | Function |
|---|---|
| Source statement | The original wording or record. |
| Evidence link | The file, commit, live-admin record, or approved note that supports the statement. |
| Meaning boundary | What the statement does not prove. |
| Label | Live verified, Merged source, Prototype, Proposed, Planned, or Not yet available. |
| Owner | The person accountable for interpretation and updates. |
| Review date | When the claim must be rechecked. |
| Contradiction log | Known conflicts that must not be silently resolved. |

## Operating Rule

Do not turn vision into implementation truth. Do not turn a prototype into an operating service. Do not turn a person signal into diagnosis, therapy, or automated guidance.
