# meUus Soul Content Map v1.0

## Purpose

This content map organizes possible future meUus Soul learning and reflection
areas before any public expansion. It provides a planning and governance
structure for content types, review status, bilingual development, and a first
safe learning slice. It does not publish, approve, or import content.

## Governing boundary

`docs/source-of-truth/soul-source-inventory-v001.md` is the governing source for
this map.

- meUus Soul is not fully rebuilt yet.
- This map is not a production claim.
- Content must pass claim review before public use.
- Archive material must be inventoried and corrected before reuse.
- A place in this map does not make content final, complete, live,
  professionally reviewed, legally reviewed, medically reviewed, or
  religious-authority reviewed.

## Content type model

| Content type | Intended function | Required boundary |
| --- | --- | --- |
| Summary page | Give a short, accessible orientation to a reviewed topic. | Must state its status and avoid presenting a summary as complete authority. |
| Deep explanation | Develop a topic through definitions, context, limitations, and related ideas. | Requires source and claim review before public use. |
| Example | Show how a principle might be understood in a bounded scenario. | Must not imply diagnosis, certainty, or a guaranteed result. |
| Q&A | Address a focused question in plain language. | Must separate education from medical, legal, religious, financial, psychological, employment, or emergency advice. |
| Glossary entry | Define one term and show how meUus currently uses it. | Must identify developing or project-specific terminology clearly. |
| Reflection exercise | Offer an optional prompt or structured pause. | Must avoid coercion, assessment claims, sensitive-data collection, and outcome promises. |
| Founder note | Preserve and present founder-authored meaning and context. | Must be attributed and distinguished from professional review or institutional authority. |
| Bangla version | Provide a meaning-preserving Bangla counterpart to reviewed content. | Requires language, context, terminology, and cultural review. |
| Review-needed note | Record why material cannot yet be treated as safe public content. | Remains internal until the identified review is completed. |
| Related path | Connect a reader to another verified or planned learning area. | Must preserve the destination's current status and not imply it is live when it is planned. |

## Status labels

| Status | Meaning |
| --- | --- |
| `Gateway only` | A public entry point exists, but the deeper destination is not complete. |
| `Draft map` | A proposed content area or structure exists only as planning. |
| `Needs claim review` | Wording, evidence, status, promises, or authority implications must be checked. |
| `Needs professional review` | Qualified review is required because the material may touch legal, medical, psychological, religious, financial, safeguarding, or other high-stakes areas. |
| `Needs translation review` | English, Bangla, terminology, meaning, accessibility, or cultural context needs review. |
| `Safe public summary` | A bounded summary has passed the required source, claim, safety, and editorial gates for its stated purpose. |
| `Deep section planned` | A deeper learning path is proposed but is not yet a live public section. |
| `Do not publish yet` | Source, rights, safety, claims, sensitivity, language, or review requirements remain unresolved. |

No content becomes `Safe public summary` merely by appearing in this map.

## Claim review checklist

Before any proposed Soul content is approved for public use, review:

- Is the statement vision or live reality, and is that distinction explicit?
- Is the material learning or advice?
- Does it imply diagnosis, assessment, scoring, or treatment?
- Does it imply religious, legal, medical, psychological, financial,
  employment, or emergency authority?
- Does it promise an outcome, job, healing, transformation, success, access,
  eligibility, or reward?
- Does it need professional review, and has the appropriate reviewer and review
  scope been recorded?
- Does it need English and Bangla meaning, terminology, or cultural review?
- Does it contain sensitive personal content or invite sensitive disclosure?
- Are source, attribution, rights, evidence, uncertainty, and current status
  recorded?
- Is the smallest accurate public claim being used?

Any unresolved high-stakes or sensitive issue keeps the content at
`Do not publish yet`.

## Bilingual structure

Future English and Bangla content should preserve meaning rather than rely on
literal translation. Each language version should retain the same status,
boundaries, uncertainty, attribution, and safety meaning.

Bangla pages may require cultural, contextual, terminology, readability, and
religious-language review beyond direct translation. English source wording may
also need correction before translation begins. Meaning differences and
founder-approved terminology should be documented instead of silently
normalized.

Bilingual parity is planned, not complete. A page in one language must not imply
that an equivalent reviewed page already exists in the other.

## First safe public Soul slice recommendation

### Knowledge Before Action — Deep Learning Path

This is the recommended first future public Soul slice because the bounded
principle already has a tracked public explanation and explicit safety limits.
The recommendation is planning only; this PR does not implement the path.

Planned pages:

1. **Overview** — define the path, audience, current status, and limits.
2. **Why it matters** — explain the value of pausing, checking knowledge, and
   matching action to evidence without promising outcomes.
3. **Example** — present a low-risk fictional scenario that separates facts,
   feelings, assumptions, uncertainty, and urgency.
4. **Reflection exercise** — offer an optional, non-diagnostic exercise that
   does not collect or require private information.
5. **Q&A** — answer common questions while preserving advice, authority,
   emergency, and uncertainty boundaries.
6. **Glossary** — define core terms such as fact, feeling, assumption,
   uncertainty, urgency, evidence, and responsible next step.
7. **Related paths** — connect only to verified public gateways or clearly
   labelled future learning areas.

Before implementation, this slice needs a source record, claim review,
editorial review, accessibility review, sensitivity check, English and Bangla
structure decision, and explicit publication status for every page.
