# Perfect Explanation Not Required

## Status Labels

- Live verified: not live verified as a public feature.
- Merged source: adjacent language exists in Journey and status boundaries, but this rule is not implemented as a named system.
- Prototype: not a working prototype.
- Proposed: proposed internal principle.
- Planned: may be used in future consented manual pilots.
- Not yet available: not an automated product capability.

## Principle

A person should not need a polished narrative before they can begin a safe, bounded conversation. Incomplete expression is allowed, but incomplete expression does not authorize the operator to make conclusions.

## What This Allows

- A short first signal.
- Uncertainty.
- A pause before explanation.
- Human clarification.
- Permission before guidance.

## What This Does Not Allow

- Diagnosis from partial language.
- Therapy claims.
- Psychological profiling.
- Automated guidance.
- Public claims based on private signals.
- Retaining sensitive data without consent and a retention rule.

## Governance Use

This principle can support humane intake design, but only if paired with boundaries, consent, deletion rights, and an evidence loop.
